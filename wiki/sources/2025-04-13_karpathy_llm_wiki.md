## 一句话摘要

Karpathy 提出了一种革命性的个人知识库模式：让 LLM 持续维护一个结构化的 Markdown Wiki，而不是每次提问时重新检索原始文档。

## 核心观点

### 问题：RAG 的局限
- 传统 RAG 每次回答都是从零开始"重新发现"知识
- 需要综合多篇文档时，模型每次都要重新找、重新拼
- 没有知识积累，答案不会随着资料增加而变得更好

### 解决方案：持久性 Wiki
- LLM 真正"读懂"资料，提取关键信息
- 将新知识**融入**已有 Wiki，而不是只是编入索引
- 更新实体页面、修订主题摘要、标注矛盾
- 交叉引用已存在，矛盾已被标记，综合结论已反映所有内容

### 系统架构

```
第一层：原始资料 (raw) — 只读的事实来源
第二层：Wiki — 由 LLM 维护的 Markdown 知识体系
第三层：Schema — 规范文档，指导 LLM 如何工作
```

### 三种核心操作

1. **录入 (Ingest)**: 资料 → 摘要 → 更新相关页面 → 更新 index → 记录 log
2. **查询 (Query)**: 问题 → 阅读相关页面 → 综合作答 → 可写回 Wiki
3. **检查 (Lint)**: 定期健康检查，排查矛盾、孤立页面、缺失引用

### 关键工具
- **Obsidian**: Wiki 的 IDE，支持双向链接和图谱视图
- **Obsidian Web Clipper**: 浏览器剪藏
- **Git**: 版本控制
- **qmd**: 本地 Markdown 搜索引擎（可选）

### 应用场景
- 个人成长（目标、健康、日记）
- 深度研究（论文、报告）
- 读书笔记（人物、主题、情节线索）
- 团队/企业 Wiki（Slack、会议记录、项目文档）

## 金句

> "维护知识库最繁琐的部分不是阅读和思考，而是记账。更新交叉引用、保持摘要时效性、维护一致性。人们放弃维护 Wiki，正是因为维护成本的增速超过了它带来的价值。LLM 解决了这个问题。"
