# 日内策略设计 Checklist

> 基于 llm-wiki 知识库中 7+篇日内策略文章的实证经验总结。每开发一个日内策略时，逐条核对。

---

## 一、数据与环境

- [ ] **数据频率确定**: 1min / 5min / 30min / tick？
  - 突破类通常需要 5min+
  - VWAP 策略通常需要 1min+
  - 半小时执行可以用 30min
- [ ] **时间范围确定**: RTH (Regular Trading Hours) 还是全天候？
  - 期货常用 RTH (9:30–16:00 ET)
  - 外汇/加密货币常用 24h
- [ ] **品种特性分析**: 该品种的日内波动率季节性如何？
  - 开盘 / 午间 / 收盘的波动率分布
  - 回撤深度偏好（浅 vs 深）— 参考 Session High Retest 的品种分裂
- [ ] **隔夜跳空处理**: 是否需要根据跳空调整参数？

---

## 二、信号设计

- [ ] **信号类型选择**: 趋势 / 回撤 / 突破 / 动量 / 事件驱动
- [ ] **信号来源确认**: 价格行为 / 指标 / 微观结构 / 外部数据
- [ ] **过滤器设计**:
  - [ ] 时间过滤（避开某些时段）
  - [ ] 波动率过滤（VIX 阈值、ATR 阈值）
  - [ ] VWAP 过滤（> VWAP 才做多，< VWAP 才做空）
  - [ ] 趋势过滤（日线级别趋势方向）
- [ ] **信号确认**: 是否需要等待下一根 bar 确认？
  - 参考 Session High Retest 的 "pullback bottomed" 检查

---

## 三、入场规则

- [ ] **入场时机**: 信号产生时立即进场 vs 等待确认？
- [ ] **入场价格**: 市价 / 收盘价 / 限价
  - 日内策略通常使用收盘价入场避免滑点
- [ ] **方向确认**: 长仅 / 短仅 / 双向？
  - 长仅策略更容易实现，但只能抓上行波动
- [ ] **执行频率**: 每小时检查一次 vs 每个 bar 检查？
  - 半小时执行可以过滤短期噪音（Systematic Intraday TF的做法）

---

## 四、出场规则

- [ ] **持有时间确定**: 固定时间 vs 信号触发 vs 固定目标
  - 日内策略常用: 持有至 RTH close / 持有至特定时间 / 无限持有
- [ ] **止损设计**:
  - [ ] 固定金额 / ATR 倍数 / VWAP / 无止损
  - [ ] 日内策略无止损持有至 close 是可行的，但需要突发性止损防止跳空
- [ ] **止盈设计**: 固定目标 / 跟踪止盈 / 时间止盈
  - 参考 VSO 策略: EOD 平仓 + VWAP 跟踪止损的组合
- [ ] **退出条件的明确性**: 什么情况下一定要退出？

---

## 五、风险管理

- [ ] **仓位大小**: 固定手数 vs 动态仓位？
  - [ ] 动态仓位是否基于当日波动率 / ATR / 目标波动率？
  - [ ] 每笔风险是否固定？（参考 Late Lunch 的 ATR 风险定位）
- [ ] **费用估算**: 佣金 + 滑点 + 点差
  - [ ] 每笔预期收益是否足以覆盖交易成本？
  - [ ] 是否在回测中包含了交易成本？（参考 Systematic Intraday TF 的 $0.0035/股）
- [ ] **过夜风险**: 是否确保每日平仓？
  - 日内策略的核心优势之一就是无过夜风险
- [ ] **最大回撤限制**: 日内策略的日内回撤可能很大，是否设置了日内最大损失限制？
- [ ] **连续亏损保护**: 是否有连续亏损后暂停交易的机制？

---

## 六、回测验证

- [ ] **回测周期**: 至少覆盖 5 年+、10 年+更佳
  - Session High Retest 使用了 17 年数据 (2008–2025)
- [ ] **多品种验证**: 单一品种的回测可能是偶然
  - 参考 Delphic Alpha 在 10 个期货品种上验证
- [ ] **步行前向验证 (Walk-Forward)**: 样本内最优参数需要步行前向验证
  - 参考 Session High Retest 的警告：walk-forward 可能降低 Sharpe 20–40%
- [ ] **交易成本敏感性分析**: 不同成本假设下表现如何变化？
- [ ] **不同市场环境**: 高波动率 / 低波动率 / 牛市 / 熊市
  - 参考 Systematic Intraday TF: VIX > 40 时 Sharpe 达 3.50
- [ ] **参数鲁棒性**: 参数在宽范围内是否都有效？
  - 参考 Systematic Intraday TF: 14 天回看有效，多种回看长度也盈利

---

## 七、实盘与执行

- [ ] **执行延迟**: 信号产生到实际执行的延迟多少？
  - 半小时执行本身就引入了延迟
  - 日内策略对延迟极敏感
- [ ] **滑点估算**: 回测是否考虑了实际滑点？
  - 参考 QuantifiedStrategies 的警告：滑点可能显著降低表现
- [ ] **资金要求**: 日内策略通常需要更高的交易频率和资金
  - Pattern Day Trader 规则 (US): 账户余额 $25,000+
- [ ] **心理准备**: 日内策略的高频交易对心理要求极高
  - 是否有自动化执行方案？
- [ ] **监控要求**: 是否需要实时监控？还是可以自动执行？
  - FDAX Session High Retest 仅 46 笔/年，监控要求低
  - NQ 同策略 830 笔/年，需要更高的自动化

---

## 八、组合与多样化

- [ ] **单一策略风险**: 是否计划将此策略作为组合的一部分？
  - 参考 Delphic Alpha 的建议："这是一个策略组合中的一部分"
- [ ] **相关性分析**: 如果有多个日内策略，它们的回撤相关性如何？
  - 参考 TradeQuantix 的经验：回撤相关性平均 ~0.34 或更低才能有效分散风险
- [ ] **日内 + 振幅 / 趋势组合**: 日内策略与振幅/趋势策略组合可以提供多样化

---

## 九、文档与维护

- [ ] **规则文档化**: 每个规则是否都有明确、可量化的定义？
- [ ] **代码备份**: 策略代码是否已经版本控制？
- [ ] **日志记录**: 是否有完整的交易日志供回测分析？
- [ ] **定期复盘**: 是否设定了定期回测验证的时间表？

---

## 十、红线检查（必须通过）

- [ ] **回测是否含有未来信息**: 检查 look-ahead bias
- [ ] **参数是否有过拟合**: 检查参数数量、优化方法
- [ ] **交易成本是否已包含**: 回测不含成本 = 不存在的策略
- [ ] **步行前向验证是否通过**: 参数在未见过的数据上是否有效
- [ ] **是否存在与交易量化的矛盾**: 策略规则是否与市场微观结构相矛盾？

## 常见日内策略设计模板

### 模板 A: 趋势突破型
```
时间框架: RTH
数据频率: 5min
信号: 价格突破动态边界
过滤器: VWAP + 波动率阈值
入场: 收盘价突破
止损: VWAP 穿越
出场: 持有至 RTH close / 跟踪止盈
仓位: 动态（ATR 或目标波动率）
```

### 模板 B: 回撤回测型
```
时间框架: RTH
数据频率: 5min
信号: 开盘涨→回撤→回测高点
过滤器: 回撤深度（根据品种选择浅/深）
入场: 回测 bar 收盘价
止损: 无 / 突发性止损
出场: 持有至 RTH close
仓位: 固定或 ATR 风险
```

### 模板 C: 新闻驱动型
```
时间框架: RTH
数据频率: 1min
信号: 新闻催化后的 VWAP + StochRSI 极端
过滤器: 新闻真实性 + 量能
入场: StochRSI 极端后反弹
止损: 前低 / 固定金额
出场: 目标价 / 时间止盈
仓位: 固定
```
