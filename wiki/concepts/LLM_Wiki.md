# LLM Wiki

**定义**: 由 LLM 持续维护的结构化 Markdown 知识库，作为原始资料和用户之间的持久性知识中间层  
**相关领域**: 知识管理、人工智能  
**首次提及**: [[2025-04-13_karpathy_llm_wiki]]  
**标签**: #llm #pkm #wiki #obsidian

## 详细解释

### 与 RAG 的区别

| | RAG | LLM Wiki |
|---|---|---|
| 知识存储 | 原始文档切片 | 结构化 Markdown 页面 |
| 查询方式 | 每次重新检索 | 直接阅读已编译的页面 |
| 知识积累 | 无 | 持续维护、交叉引用、标注矛盾 |
| 维护成本 | 人工维护难度高 | LLM 自动维护 |

### 主要特点
1. **持久性** — 知识只需编译一次，之后持续更新
2. **结构化** — 实体页面、概念页面、综合结论分类明确
3. **双向链接** — 页面之间相互引用，支持图谱探索
4. **更新式** — 新资料录入时，LLM 会更新已有相关页面

### 三层架构
1. **原始资料层** — raw/，只读
2. **Wiki 层** — wiki/，由 LLM 维护
3. **Schema 层** — AGENTS.md/CLAUDE.md，规范约定

### 三种操作
- **Ingest** — 录入新资料
- **Query** — 向知识库提问
- **Lint** — 定期健康检查

## 与其他概念的关系
- vs [[RAG]]: RAG 是动态检索，LLM Wiki 是静态编译
- + [[Obsidian]]: Obsidian 是 LLM Wiki 的理想载体
- + [[Memex]]: 精神上一脉相承，LLM 解决了 Memex 缺少维护者的问题

## 应用实例
- 个人成长追踪（日记、目标、健康）
- 深度研究某个课题
- 读书笔记管理
- 团队内部知识库
